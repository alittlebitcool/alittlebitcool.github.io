设计模式总结-结构型

发表于2025-03-04|更新于2025-03-06
|浏览量:

1. 适配器模式（Adapter Pattern）

将一个类的接口转换成客户端期望的另一个接口，使原本不兼容的类可以一起工作。

套路

  • 定义一个目标接口（Target）。
  • 创建一个适配器类（Adapter），实现目标接口并持有被适配者（Adaptee）的引用。
  • 在适配器类中调用被适配者的方法。

2. 装饰器模式（Decorator Pattern）

目的：动态地为对象添加额外的职责，而不改变其结构。

套路

  • 定义一个组件接口（Component）。
  • 创建具体组件类（ConcreteComponent）。
  • 创建装饰器基类（Decorator），持有组件接口的引用。
  • 在具体装饰器类（ConcreteDecorator）中添加额外的行为。

3. 代理模式（Proxy Pattern）

为其他对象提供一个代理，以控制对这个对象的访问。

套路

  • 定义一个主题接口（Subject）。
  • 创建真实主题类（RealSubject）。
  • 创建代理类（Proxy），持有真实主题的引用，并在其方法中添加额外的控制逻辑。

4. 外观模式（Facade Pattern）

目的：为子系统中的一组接口提供一个统一的接口，简化客户端的使用。

套路

  • 定义多个子系统类（SubsystemA、SubsystemB 等）。
  • 创建外观类（Facade），封装子系统的调用逻辑。
  • 客户端通过外观类与子系统交互。

5. 组合模式（Composite Pattern）

目的：将对象组合成树形结构以表示“部分-整体”的层次结构。

模板

java

Copy

套路

  • 定义一个组件接口（Component）。
  • 创建叶子节点类（Leaf）和复合节点类（Composite）。
  • 复合节点类可以包含其他组件（叶子节点或复合节点）。

6. 桥接模式（Bridge Pattern）

目的：将抽象部分与实现部分分离，使它们可以独立变化。

套路

  • 定义一个实现接口（Implementor）。
  • 创建具体实现类（ConcreteImplementorA）。
  • 定义抽象类（Abstraction），持有实现接口的引用。
  • 扩展抽象类（RefinedAbstraction）以实现具体逻辑。

7. 享元模式（Flyweight Pattern）

目的：通过共享技术有效地支持大量细粒度对象。

套路

  • 定义一个享元接口（Flyweight）。
  • 创建具体享元类（ConcreteFlyweight），区分内部状态和外部状态。
  • 使用享元工厂（FlyweightFactory）管理共享的享元对象。
